🎨 ITAM Shell Template

A ready-to-use Zendesk navigation shell for prototyping IT Asset Management experiences

Employee Service Design Kit

What is this?

The ITAM Shell Template is a pre-built navigation shell with Zendesk's Admin Center and Support chrome (header + sidebar). It lets you focus on building your content without recreating navigation from scratch.

💡 Key concept: The template provides the shell (navigation). You build the content (your pages). Everything you create goes inside the <Main> component.

✅ What's included

  • Full header with product tray
  • Left sidebar navigation
  • Admin Center page example
  • Support page example
  • Flora design system
  • All spacing and typography

✅ What you build

  • Your page content
  • Tables, forms, lists
  • Custom components
  • Data and logic
  • Your prototype features

Quick start (Cursor workflow)

Follow these steps to create your first prototype:

1

Clone the design kit (one-time only)

mkdir -p ~/zendesk/design-kits ~/zendesk/prototypes
cd ~/zendesk/design-kits
git clone git@github.com:zendesk/employee-service-design-kit.git

You only do this once! The design kit stays there forever.

2

Create your project folder

mkdir ~/zendesk/prototypes/my-new-project

Use descriptive names like asset-fields-v2 or status-redesign.

3

Open the folder in Cursor

In Cursor:

  • Click File > Open Folder
  • Navigate to ~/zendesk/prototypes/my-new-project
  • Click Open
4

Open Cursor's integrated terminal

Press Ctrl + ` or click Terminal > New Terminal

5

Run the setup script

~/zendesk/design-kits/employee-service-design-kit/scripts/setup-here.sh itam-shell

This copies the template, installs dependencies, and initializes git. Takes ~30 seconds.

6

Start the dev server

npm run dev

Open http://localhost:5173 in your browser.

🎉 Done! Your prototype is running with full Zendesk navigation and Flora styling.

Understanding the <Main> content area

The <Main> component is where all your prototype content lives. Think of it as the white box in the center of the page.

Visual breakdown

← Header (fixed, don't change)
<Main> content area
← This is where YOUR content goes
← Change everything here!

What goes in <Main>?

  • Your page components - Tables, forms, cards, lists
  • Breadcrumbs - Navigation trail
  • Page titles and descriptions - H1 headings, supporting text
  • All your content - Everything users interact with

Spacing rules for <Main>

  • Container padding: 20px top/bottom, 40px left/right
  • Page title: No top margin (starts at container edge)
  • Header to content: 32px separation
  • Max width: 1200px for readability

Code example

<Main id="main-content">
  <div style={{ maxWidth: 1200, padding: '20px 40px' }}>
    <h1>My page title</h1>
    <p>Description text</p>

    <div style={{ marginTop: 32 }}>
      {/* Your content here */}
    </div>
  </div>
</Main>

Product switcher (Admin Center ↔ Support)

The template includes navigation for both Admin Center and Support. Users can switch between them using the product tray.

How it works

  1. Click the product tray button (top-left, says "Admin center")
  2. Select Support from the dropdown
  3. The page switches to Support view with different navigation
  4. Click again to switch back to Admin Center
💡 Pro tip: You have separate page files for each product:
src/pages/FieldOrder.tsx (Admin Center)
src/pages/SupportFieldOrder.tsx (Support)

Only need one product? Disable the switcher

If you're only prototyping Admin Center or Support (not both), you can remove the switcher:

Option 1: Remove the product tray completely

In src/App.tsx, change line 83:

// Before
<Product key={currentProduct} locale="en-US" isFlora products={products}>

// After (no products prop = no product tray)
<Product key={currentProduct} locale="en-US" isFlora>

Option 2: Lock to one product

In src/App.tsx, remove the switcher logic:

// Remove useState and product switching
// Just render one view:
<Product locale="en-US" isFlora>
  <AppHeader product="admin-center" />
  <Sidebar />
  <Main id="main-content">
    <FieldOrder />
  </Main>
</Product>
✨ Tip: Most ITAM prototypes need both products, so the switcher is useful for showing the full experience.

What to change vs. what to keep

✅ Change these

  • src/pages/
  • src/components/content/
  • src/data/
  • Content inside <Main>

This is YOUR prototype content.

❌ Don't change these

  • src/components/shared/
  • Header.tsx
  • Sidebar.tsx
  • <Product> shell in App.tsx

This is the navigation shell.
